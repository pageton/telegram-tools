# Design Context

## Users

Telegram developers, security researchers, and library maintainers who work with MTProto session strings. They arrive with a specific task — generate, convert, analyze, or test — and want to leave quickly. Technical power-users who value speed and clarity over hand-holding. They may be debugging auth issues, migrating between client libraries, or inspecting session integrity.

## Brand Personality

**Precise, technical, trustworthy.**

The interface should feel like a well-made developer tool — confident and exact, never flashy or attention-seeking. Every element earns its place. The experience should inspire trust: "this tool handles my credentials and auth keys correctly." Think Linear's clarity and Vercel's polish — dark, spacious, subtly layered, with crisp typography and purposeful motion.

## Aesthetic Direction

- **Theme**: Dark-only. Deep navy/indigo base (`surface-950`) with indigo primary accents and sky-blue secondary accents.
- **Tone**: Refined dark UI with subtle depth — glassmorphism and blur effects used sparingly and purposefully, never decoratively.
- **Typography**: Inter for body, JetBrains Mono for code/session strings. Clean hierarchy with restrained weight variation.
- **References**: Linear (spacious layouts, subtle borders, purposeful whitespace), Vercel Dashboard (depth through layering, confident restraint).
- **Anti-references**: No cryptic hacker-terminal aesthetic. No Matrix green-on-black. No playful mascots or toy-like elements. No generic SaaS marketing fluff. Avoid overused AI aesthetics: cyan-on-dark, purple-to-blue gradients, neon accents, gradient text.

## Design Principles

1. **Precision over personality** — Every pixel is intentional. No decorative elements that don't serve clarity or trust. The tool speaks through accuracy, not flash.

2. **Speed of comprehension** — Information architecture mirrors the user's mental model. Session formats, DC numbers, auth keys — presented in scannable, structured layouts with clear visual hierarchy.

3. **Confident restraint** — Like Linear and Vercel, the design earns sophistication through what it omits. Generous whitespace, subtle borders, restrained color. Depth through layering, not decoration.

4. **Trust through consistency** — Uniform patterns across all five tools. Same border radius, same spacing rhythm, same color logic. Consistency signals reliability — critical when handling auth credentials.

5. **Motion as feedback, not entertainment** — Animations convey state changes (loading, success, error, transitions between tools). Exponential easing, no bounce. Respects `prefers-reduced-motion`.

## Technical Constraints

- **Framework**: Svelte 5 with runes (`$state`, `$derived`), no component library
- **Styling**: Tailwind CSS v4 with custom `@theme` tokens (defined in `src/app.css`)
- **Deployment**: Static site on GitHub Pages (`/telegram-tools/` base path)
- **Browser-only**: All crypto/auth happens client-side — no backend
- **Fonts**: Inter (body) + JetBrains Mono (code) via Google Fonts
- **Accessibility**: Must support `prefers-reduced-motion`. Keyboard-navigable forms. Sufficient contrast on dark backgrounds.

## Color System

- **Primary**: Indigo spectrum (`#6366f1` → `#1e1b4b`) — actions, focus, emphasis
- **Surface**: Slate spectrum (`#f8fafc` → `#020617`) — backgrounds, containers, borders
- **Accent**: Sky blue (`#0ea5e9`) — secondary actions, info states
- **Semantic**: Green success, Red error, Amber warning — status indicators only
- **Never**: Pure black (#000) or pure white (#fff) — always tinted neutrals
